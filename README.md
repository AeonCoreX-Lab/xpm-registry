# xpm-registry

**The official package registry for the X-Phage language**

Fully serverless вЂ” runs entirely on GitHub infrastructure.
No registry server. No fees. No maintenance.

---

## Publishing a package

```bash
export GITHUB_TOKEN=ghp_xxxxxxxxxx   # repo scope
xpm publish
```

**That's it.** XPM automatically:
1. Creates a GitHub Release in this repo
2. Uploads your `.xh` file as a release asset
3. Generates and uploads a SHA256 checksum
4. Updates the sparse index (`index/<prefix>/<n>.toml`)

No `gh` CLI, no manual steps.

---

## Installing a package

```bash
xpm install net-http          # latest
xpm install net-http@^1.2     # semver range
xpm install data-json@2.0.0   # exact version
```

## Index format (TOML)

Index files live at `index/<first-2-chars>/<name>.toml`.

Example: `index/ne/net-http.toml`

```toml
# net-http package index
# Generated by XPM вЂ” do not edit manually.

[[versions]]
version    = "1.2.0"
checksum   = "sha256:abc123..."
xh_url     = "https://github.com/AeonCoreX-Lab/xpm-registry/releases/download/net-http@1.2.0/net-http.xh"
sha256_url = "https://github.com/AeonCoreX-Lab/xpm-registry/releases/download/net-http@1.2.0/net-http.sha256"
yanked     = false
published  = "2025-01-01T00:00:00Z"

  [[versions.deps]]
  name     = "data-json"
  req      = "^2.0"
  optional = false
```

Index is served via `raw.githubusercontent.com` вЂ” CDN-backed, no rate limits, no auth needed.

## Version constraints

| Syntax | Meaning |
|--------|---------|
| `1.2.0` | Exact version |
| `^1.2.0` | в‰Ґ1.2.0, <2.0.0 |
| `~1.2.0` | в‰Ґ1.2.0, <1.3.0 |
